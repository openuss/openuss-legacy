Default Interaction Buttons

Some View Managers may use the Default Element Interaction Buttons provided by the platform for standard tasks like navigating through the Structure Tree or using the Split Screen Mode. As Default Element Interaction Buttons you may find the following placed at the left side of the Interaction Panel:

 Split Screen Mode Enable or disable (Split Screen Mode)
Switch panels in Split Screen Mode Switch between primary and secondary Content Panel
Structure Tree navigation Move forward and backward through the elements of the Structure Tree
Full Screen Mode Switch between Full Screen and windowed mode
Scale Mode Switch original or panel scaled size mode

Which buttons actually are displayed depends upon if the selected View Manager considers this as suitable.

View Specific Element Interaction Buttons

Beside the Default Interaction Buttons, any View Manager is free to place additional Interaction Buttons into the Interaction Panel to enable view specific interaction features. These buttons are called View Specific Interaction Buttons and may be unique for different Views. Examples could be a play or stop button to interact with multimedia content. Please refer to the according View Manager Documentation about their specific features.
